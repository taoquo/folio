# Web Reading

Use this guide for content-heavy browser pages: long articles, reports, case studies, research notes, release stories, manifestos, and document-to-Web publishing.

Read `web-foundation.md` first.

## Core Job

A reading page should let the reader understand the argument, locate evidence, and keep orientation while scrolling.

It is not a landing page unless the user explicitly asks for one.

## Standard Structure

1. Opening title block: title, subtitle, author/date/source context.
2. Optional contents rail: useful for long pages with three or more major sections.
3. Claim-led sections: each section opens with the argument, then evidence.
4. Figures: every figure needs a caption that states the insight.
5. Source treatment: source rail, footnotes, or reference list.
6. Reading progress: use when the page is long enough that orientation matters.

## Layout Patterns

### Single Column With Source Rail

Use for essays and research notes.

- Main column: 660-760px reading measure.
- Source rail: citations, metadata, related links, or section anchors.
- Mobile: source rail moves below the related paragraph or into collapsible notes.

### Editorial Split

Use for reports and case studies.

- Left rail: contents or chapter markers.
- Main column: prose, figures, tables.
- Right rail: optional sources, definitions, or status.

### Figure-Led Section

Use when the visual evidence carries the section.

- Claim appears before the figure.
- Figure appears at a stable aspect ratio.
- Caption explains the insight, not just the chart title.
- Supporting prose follows the figure.

### Case Study Sequence

Use for project narratives.

- Problem and stakes.
- Decision points.
- Evidence.
- Outcome.
- Reflection or next step.

## Motion Patterns

Use motion sparingly:

- section reveal on first viewport entry
- active TOC marker while scrolling
- reading progress line
- figure/caption reveal as one unit
- source note highlight when a citation is clicked

Avoid continuous animation while the reader is in the body text.

## Content Rules

- Headings should read as a summary of the argument.
- Long pages need visible structure before decorative imagery.
- Pull quotes should clarify a claim, not interrupt evidence.
- Links and citations must remain visible and readable.
- If a document already has a strong PDF hierarchy, preserve its section logic rather than inventing a marketing narrative.

## Anti-Patterns

- replacing the argument with a marketing hero
- hiding sources in tiny footer text
- repeated decorative cards for every paragraph
- huge quote blocks with weak evidence
- scroll effects that move text while it is being read
- dark atmospheric sections that make long reading harder
